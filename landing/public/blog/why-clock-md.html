Why clock.md exists

2026-02-10

AI agents often “know” facts but still miss the most basic temporal context: what time it is, what day it is, how long it has been since the last message, whether a deadline is in the past or the future, and which timezone the user is operating in.

clock.md exists to make that failure mode explicit and fixable: if time is unknown, the agent should ask; if time is known, the agent should label it, store it, and reason about it safely.
